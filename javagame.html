Java Script Game

Main Idea Of this project



The Quiz Game can be made using JS using various approaches and some of the approaches and their functionalities are mentioned bellow. You can develop your game using these approaches or if you have any different approach in the mind you can discuss it with you mentor and can develop it.


Approach 1

In this approach of the game the user gets a list of question and can answer in whichever order he wants (which means the user can surf around the questions) and at the submission or completion of the quiz, the result will be shown.


Functionalities

  1.  The first basic functionality of the game is to have a home page from where the user can select the genre of the quiz whichever he/she wants to play.
  2.  Once the user selects the genre, the user must be transferred to the Quiz Start page which asks for the player’s name and a start button.
  3.  On the click of the start button the user must be asked for reconfirmation of the starting the quiz and on getting the positive response the user must be transferred to the question answers page.
  4.  It is preferable that you generate individual pages for individual questions rather than displaying all the questions on a single page. (Try to use some data structure to store these questions and try not to hardcode the questions.)
  5.  Each question page should have its respective options and the following buttons:
    Previous: To redirect the user to the previous question.(This button should be inactive on the first question)
    Next: To redirect the question to the next question.(This button should be inactive on the last question)
    Submit: To submit the quiz and end it.
  6.  Try to add the functionality of the timer in it.
  7.   At the end once the user submits the quiz it should get the report of the quiz played which shows total attempted questions, correctly answered, and wrongly answered and the score calculating all of it.
  8.   At the end there should be a button to redirect to the home page of the game.
  9.  Try to maintain a record of the players and their scores according to the genre which can be showed when the user submits the quiz to show the ranking of the user.
  10.  Make sure the evaluation of the questions is done correctly.


Approach 2


In this approach, the game would be level oriented. There would be a certain number of levels and the user need to complete all those level to win the game. At every level the user needs to answer the question and if the answer is correct the user gets promoted to the next level.( You can decide the number of levels and number of questions in each level according to your convenience).On wrong response to the question the game gets over and need to start again from the beginning.

Functionalities

  1.  The first basic functionality of the game is to have a home page from where the user can select the genre of the quiz whichever he/she wants to play.
  2.  Once the user selects the genre, the user must be transferred to the Quiz Start page which asks for the player’s name and a start button.
  3.  On the click of the start button the user must be asked for reconfirmation of the starting the quiz and on getting the positive response the user must be transferred to the Level 1 page.
  4.  If the response of the question asked in Level 1 is correct the user should be moved to the Level 2 page and if the response is wrong the game gets over and the user gets redirected to the home page by showing appropriate message of game over.
  5.  On the correct response of each question the Level of the user should be increased each time and when the user surpasses the highest level, he should be declared winner and transformed to the Winner page and then to the home page.
  6.  Make sure the evaluation is done correctly at every level.


Note: Implementing all the functionalities is not compulsory but implement as many as you can as it will help you in learning more and getting more comfortable with HTML,CSS and JS.
