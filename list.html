TO Do List

Main Idea Of this project

The to-do list will comprise of a variety of tasks to be performed by the programmer to implement such as


  1.     Maintaining a number of lists created by the user (Yes, we are considering only one user as of now)
  2.     Allowing the user to create, check-off items or delete items in the list.



Functionality :

  1.    The basic functionality of any to-do list is to provide the user with a detailed list of tasks assigned to him/her for that particular day.
  2.   The first basic functionality of this application is to have a homepage from where the user can select a particular date for which he/she wishes to view or to modify the to-do list assigned for that date.
  3.    Please note that once the date is selected the user must be transferred to a new page containing the details of the to-do list for the selected date. This must be accomplished by redirecting the user to the destination page.
  4.   The second and the most important functionality is the actual to-do list information for a particular date. Once the user is redirected to this page from the home-page he/she must be able to view the available tasks as well as insert new tasks and delete/strike the completed tasks.
  5.    Insert new task command button needs to be available on this page, when it is selected the user should be able to provide information regarding 2 major fields… ‘what is the task needed to be done ‘ and the ‘due date’ till which the task needs to be completed. Once all the details of the new task have been completed that task details needs to be added to the other tasks that are required to be executed on that particular date.
  6.   These are the basic functionalities that are needed for the to-do list. Please feel free to add any extra functionalities like sub-fields / sub-entries to a particular task and innovating your to-do list as you like.
